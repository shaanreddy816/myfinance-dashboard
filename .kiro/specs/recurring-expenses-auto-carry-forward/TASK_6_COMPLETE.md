## 📊 RecurringExpenseEngine Implementation

### Core Architecture

The engine follows a multi-step execution flow:

```
1. Get Active Master Expenses
   ↓
2. Generate Monthly Entries (with duplicate check)
   ↓
3. Check and Complete EMIs
   ↓
4. Send Subscription Renewal Reminders
   ↓
5. Log Execution Results
```

### Main Methods

#### 1. executeMonthlyCarryForward()
**Purpose**: Main scheduler - runs on first day of month

**Features**:
- Executes all carry-forward operations
- Tracks comprehensive metrics
- Handles errors gracefully
- Logs execution to database
- Returns execution result

**Metrics Tracked**:
- `execution_date` - Date of execution
- `execution_time` - Timestamp
- `status` - 'success', 'partial_success', or 'failed'
- `entries_generated` - Count of entries created
- `emis_completed` - Count of EMIs marked as completed
- `reminders_sent` - Count of renewal reminders sent
- `errors_encountered` - Count of errors
- `error_details` - Array of error information
- `execution_duration_ms` - Execution time in milliseconds

**Example**:
```javascript
// Run the monthly carry-forward
const result = await recurringExpenseEngine.executeMonthlyCarryForward();

console.log('Status:', result.status);
console.log('Entries generated:', result.entries_generated);
console.log('EMIs completed:', result.emis_completed);
console.log('Errors:', result.errors_encountered);
console.log('Duration:', result.execution_duration_ms, 'ms');
```

**Expected Output**:
```

[RecurringExpenseEngine] Status: success
[RecurringExpenseEngine] Entries: 3, EMIs completed: 1, Errors: 0
```

#### 2. getActiveMasterExpenses(currentDate)
**Purpose**: Query all active master expenses that should generate entries

**Features**:
- Queries master_expenses with status='active'
- Filters by end_date (null or >= current date)
- Returns array of MasterExpense instances

**Logic**:
```javascript
// Active if:
// 1. status = 'active' AND
// 2. (end_date IS NULL OR end_date >= current_date)
```

**Example**:
```javascript
const activeExpenses = await recurringExpenseEngine.getActiveMasterExpenses(new Date());
console.log('Active expenses:', activeExpenses.length);
```

#### 3. generateMonthlyEntry(masterExpense, month, year)
**Purpose**: Generate a single monthly expense entry from a master expense

**Features**:
- Checks for duplicate entries (prevents re-generation)
- Sets entry_date to first day of month
- Uses ExpenseEntryService.createAutoGenerated()
- Logs success/failure

**Duplicate Prevention**:
```javascript
// Checks if entry already exists for this master expense + month
const existing = await sb
    .from('expenses')
    .select('id')
    .eq('master_expense_id', masterExpense.id)
    .eq('entry_date', '2024-12-01')
    .maybeSingle();

if (existing) {
    // Skip - already generated
    return;
}
```

**Example**:
```javascript
const masterExpense = new MasterExpense({
    id: 'master-uuid-123',
    name: 'Home Loan EMI',
    amount: 35000,
    category: 'housing'
});

await recurringExpenseEngine.generateMonthlyEntry(masterExpense, 12, 2024);
// Creates expense entry with entry_date = '2024-12-01'
```

#### 4. checkAndCompleteEMIs(currentDate)
**Purpose**: Find EMIs that have reached their end date and mark as completed

**Features**:
- Queries EMIs with is_emi=true, status='active'
- Checks if end_date < current_date
- Updates status to 'completed'
- Sends completion notification
- Returns count of completed EMIs

**Logic**:
```javascript
// For each active EMI:
if (emi.end_date < currentDate) {
    // Mark as completed
    // Send notification
}
```

**Example**:
```javascript

console.log('EMIs completed:', completedCount);
```

**Notification Output**:

```
  Total paid: ₹900000
  Completion date: 2024-12-01
```

#### 5. sendRenewalReminders(currentDate)
**Purpose**: Send reminders for subscriptions expiring within 7 days

**Features**:

- Sends reminder for each subscription

**Logic**:

### Error Handling Flow

```javascript
// If errors occur during execution:

{
    status: 'partial_success', // or 'failed'
    entries_generated: 3,
    errors_encountered: 2,
    error_details: [
        {
            master_expense_id: 'uuid-123',
            master_expense_name: 'Subscription X',
            error: 'Database connection timeout'
        },
        {
            master_expense_id: 'uuid-456',
            master_expense_name: 'EMI Y',
            error: 'Validation failed'
        }
    ]
}
```

**Error Handling Strategy**:
1. Errors are caught per master expense (one failure doesn't stop others)
2. Successful entries are committed even if some fail
3. All errors are logged with details
4. Status reflects partial success vs complete failure
5. Execution continues to completion

### Option 2: Browser-Based Scheduler (Simple)

```javascript
// Check on app load if it's the first day of the month
async function checkAndRunCarryForward() {
    const today = new Date();
    const dayOfMonth = today.getDate();
    
    if (dayOfMonth === 1) {
        // Check if already run today
        const lastExecution = await recurringExpenseEngine.getLastExecution();
        const lastRunDate = lastExecution ? lastExecution.execution_date : null;
        const todayDate = today.toISOString().split('T')[0];
        
        if (lastRunDate !== todayDate) {
            console.log('Running monthly carry-forward...');
            await recurringExpenseEngine.executeMonthlyCarryForward();
        }
    }
}

// Call on app initialization
checkAndRunCarryForward();
```

### Option 3: Serverless Function (Recommended for Production)

```javascript
// Vercel Serverless Function: api/cron/carry-forward.js

export default async function handler(req, res) {
    // Verify cron secret for security
    if (req.headers['authorization'] !== `Bearer ${process.env.CRON_SECRET}`) {
        return res.status(401).json({ error: 'Unauthorized' });
    }
    
    // Initialize services
    const sb = createClient(process.env.SUPABASE_URL, process.env.SUPABASE_SERVICE_KEY);
    const masterExpenseService = new MasterExpenseService(sb);
    const expenseEntryService = new ExpenseEntryService(sb);
    const engine = new RecurringExpenseEngine(masterExpenseService, expenseEntryService, sb);
    
    // Execute carry-forward
    const result = await engine.executeMonthlyCarryForward();
    
    return res.status(200).json(result);
}
```
