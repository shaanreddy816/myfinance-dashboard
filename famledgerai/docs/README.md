## 🤝 Contributing to Documentation

When adding new documentation:

1. **Choose the right folder** based on content type
2. **Use descriptive filenames** in UPPERCASE with underscores
3. **Include a header** with title, date, and author
4. **Update the index** in the respective folder's README
5. **Link related docs** for easy navigation

### Documentation Standards

- Use Markdown format (.md)
- Include table of contents for long documents
- Add code examples where applicable
- Use emojis for visual hierarchy
- Keep language clear and concise
- Update this README when adding new categories
