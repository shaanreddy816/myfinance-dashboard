# Recurring Expenses Feature - Database Setup Guide

## Overview

This guide walks you through setting up the database schema for the Recurring Expenses & Auto-Carry Forward feature. This feature enables automatic monthly expense generation from master expense definitions.

- Access to your Supabase project dashboard
- SQL Editor access in Supabase
- Basic understanding of PostgreSQL

## Database Schema Overview

### Table: master_expenses

Stores recurring expense definitions (EMIs, subscriptions, utilities, groceries).

**Key Columns:**
- `id` - UUID primary key
- `family_id` - User email (acts as family identifier)
- `name` - Expense name (e.g., "Home Loan EMI")
- `amount` - Monthly amount
- `category` - Expense category
- `expense_type` - Type: 'emi', 'subscription', 'utility', 'grocery', 'other'
- `start_date`, `end_date` - Date range (end_date NULL = indefinite)
- `status` - Lifecycle: 'active', 'paused', 'completed', 'cancelled'
- `is_emi` - Boolean flag for EMI expenses
- `tenure_months` - EMI tenure in months
- `is_subscription` - Boolean flag for subscriptions
- `renewal_date` - Subscription renewal date

### Table: expenses

Stores individual expense entries (both manual and auto-generated).

**Key Columns:**
- `id` - UUID primary key
- `family_id` - User email
- `master_expense_id` - Reference to master_expenses (NULL for manual entries)
- `amount` - Expense amount
- `category` - Expense category
- `entry_date` - Date of expense
- `auto_generated` - Boolean: true if generated by carry-forward engine
- `manually_adjusted` - Boolean: true if user edited an auto-generated entry
- `original_amount` - Original amount before manual adjustment

### Table: recurring_engine_logs

Tracks execution history of the monthly carry-forward engine.

**Key Columns:**
- `id` - UUID primary key
- `execution_date` - Date of execution
- `status` - 'success', 'partial_success', 'failed'
- `entries_generated` - Count of entries created
- `emis_completed` - Count of EMIs marked as completed
- `errors_encountered` - Count of errors
- `error_details` - JSONB with error information

## Troubleshooting

### Issue: RLS Policies Blocking Queries

If you get permission errors when querying:

1. Check if RLS is enabled:
```sql
SELECT tablename, rowsecurity 
FROM pg_tables 
WHERE schemaname = 'public' 
    AND tablename IN ('master_expenses', 'expenses', 'recurring_engine_logs');
```

2. Temporarily disable RLS for testing (NOT recommended for production):
```sql
ALTER TABLE master_expenses DISABLE ROW LEVEL SECURITY;
ALTER TABLE expenses DISABLE ROW LEVEL SECURITY;
```

3. Re-enable after testing:
```sql
ALTER TABLE master_expenses ENABLE ROW LEVEL SECURITY;
ALTER TABLE expenses ENABLE ROW LEVEL SECURITY;
```

### Issue: Foreign Key Constraints

If you have existing `families` or `users` tables with UUID primary keys, update the migration script:

```sql
-- Replace VARCHAR(255) with UUID and add foreign key constraints
family_id UUID NOT NULL REFERENCES families(id) ON DELETE CASCADE,
created_by UUID NOT NULL REFERENCES users(id),
```

### Issue: Duplicate Table Names

If tables already exist, you can drop them first (WARNING: This deletes all data):

```sql
DROP TABLE IF EXISTS recurring_engine_logs CASCADE;
DROP TABLE IF EXISTS expenses CASCADE;
DROP TABLE IF EXISTS master_expenses CASCADE;
```

Then re-run the migration script.

## Testing Queries

### Query Active Master Expenses

```sql
SELECT 
    name, 
    amount, 
    category, 
    expense_type, 
    status,
    CASE 
        WHEN end_date IS NULL THEN 'Indefinite'
        ELSE end_date::text
    END as end_date
FROM master_expenses
WHERE family_id = 'bhatinishanthanreddy@gmail.com'
    AND status = 'active'
ORDER BY created_at DESC;
```

### Query Monthly Expenses with Source

```sql
SELECT 
    e.entry_date,
    e.amount,
    e.category,
    e.auto_generated,
    e.manually_adjusted,
    m.name as master_expense_name
FROM expenses e
LEFT JOIN master_expenses m ON e.master_expense_id = m.id
WHERE e.family_id = 'bhatinishanthanreddy@gmail.com'
    AND e.entry_date >= DATE_TRUNC('month', CURRENT_DATE)
    AND e.entry_date < DATE_TRUNC('month', CURRENT_DATE) + INTERVAL '1 month'
ORDER BY e.entry_date DESC, e.created_at DESC;
```

### Calculate Total Monthly Recurring Obligations

```sql
SELECT 
    SUM(amount) as total_monthly_recurring,
    COUNT(*) as active_master_expenses
FROM master_expenses
WHERE family_id = 'bhatinishanthanreddy@gmail.com'
    AND status = 'active'
    AND (end_date IS NULL OR end_date >= CURRENT_DATE);
```

### Query EMIs with Remaining Tenure

```sql
SELECT 
    name,
    amount as monthly_emi,
    start_date,
    end_date,
    tenure_months,
    total_amount,
    EXTRACT(YEAR FROM AGE(end_date, CURRENT_DATE)) * 12 + 
    EXTRACT(MONTH FROM AGE(end_date, CURRENT_DATE)) as remaining_months,
    amount * (EXTRACT(YEAR FROM AGE(end_date, CURRENT_DATE)) * 12 + 
              EXTRACT(MONTH FROM AGE(end_date, CURRENT_DATE))) as remaining_amount
FROM master_expenses
WHERE family_id = 'bhatinishanthanreddy@gmail.com'
    AND is_emi = true
    AND status = 'active'
ORDER BY end_date;
```
